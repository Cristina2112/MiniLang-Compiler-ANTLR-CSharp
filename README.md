# MiniLang Compiler

![C#](https://img.shields.io/badge/Language-C%23-blue) ![ANTLR4](https://img.shields.io/badge/Tool-ANTLR4-orange) ![Semantic Analysis](https://img.shields.io/badge/Feature-Type%20Checking-green)

A custom compiler frontend developed for a C-like procedural mini-language (`MiniLang`).
This project demonstrates the complete compilation pipeline: **Lexical Analysis**, **Syntactic Parsing** (using ANTLR4), and **Semantic Validation** via the Visitor pattern.

##  Key Features

### 1. Grammar Definition (ANTLR4)
* Defined a robust context-free grammar (`MiniLang.g4`) supporting:
  * **Control Structures:** `if-else`, `while`, `for` loops.
  * **Functions:** Definition, parameters, return types, and recursive calls.
  * **Variables:** Global/Local scope, constants (`const`), and primitive types (`int`, `float`, `string`, `void`).
  * **Operators:** Arithmetic, relational, logical, and assignment operators with correct precedence.

### 2. Compilation Phases
* **Lexical Analysis:** Tokenizes the source code into valid lexemes, handling keywords, identifiers, literals, and comments.
* **Syntactic Analysis (Parsing):** Generates the Abstract Syntax Tree (AST) based on the grammar rules defined in `.g4`.
* **Semantic Analysis:** Implements a custom **Visitor** to traverse the AST and enforce language rules:
  * **Symbol Table:** Manages variable and function scopes, ensuring unique declarations and resolving identifiers.
  * **Type Checker:** Validates type compatibility (e.g., prevents assigning a `string` to an `int`) and verifies function return types.
  * **Scope Validation:** Detects duplicate variables, undefined identifiers, and parameter conflicts.

### 3. Error Handling
* Catches and reports syntax errors (from ANTLR) and semantic errors (e.g., "Type mismatch", "Variable not declared") with precise line numbers in an output file.

##  Tech Stack & Architecture

* **Language:** C# (.NET Framework)
* **Parser Generator:** ANTLR4 (Another Tool for Language Recognition)
* **Design Patterns:** Visitor Pattern (for AST traversal)

## Project Structure

* `MiniLang.g4` - The ANTLR grammar file defining the language syntax.
* `CompilerVisitor.cs` - Core logic traversing the AST to validate semantics and build the symbol table.
* `SymbolTable.cs` - Manages scopes, global/local variables, and function signatures.
* `TypeChecker.cs` - Logic for validating mathematical and logical type consistency.
* `MiniLangLexer.cs` & `MiniLangParser.cs` - Generated by ANTLR to handle tokenization and parsing.
* `Program.cs` - Main entry point orchestrating the compilation pipeline and file I/O.
